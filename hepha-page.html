This is the page containing information and documentation on Hepha Game Engine.

Documentation

How to set up your project
To set up your project, first download the Hepha Game Engine here. Once you have downloaded Hepha Engine, take some time to see if you can figure some of the functions out yourself, otherwise, initialize an object where you would like inside of your main.cpp file.

How to Initialize Objects
Try setting up your main.cpp as following: 

                
                    #include "Application.h"

                    Hepha::Rect shape;

                    Hepha::GameObject exampleObject("Sprite1", shape, 
                    glm::vec2(.2f), glm::vec2(0.0f, 0.0f), glm::vec3(1.0f), 
                    "Tex/Kirby.png", Hepha::Texture::CLOSEST);

                    Hepha::Application::Application()
                    {
                        //initialize your objects here
                        exampleObject.SetPosition(glm::vec2(6.0f, 1.0f));
                    }

                    void Hepha::Application::Update()
                    {
                        exampleObject.Update();
                        //your game logic
                    }

                    void Hepha::Application::Render()
                    {
                        exampleObject.Render();
                        //render your objects here
                    }

                    int main()
                    {

                        Application app;

                        while (app.getWinOpen())
                        {
                            app.Render();
                            app.Update();
                        }
    
                        glfwTerminate();
                        return 0;
                    }

In this example, we put our game logic in the main.cpp file, you could also create a file with the name of your choice and do this. Just make sure to include Application.h. Regarding the objects, there are several ways you can initialize them, but the one in the example is the most flexible because of it's multiple parameters.

The first two parameters are object name and what shape the object is. The object can be named anything you would like, but make sure to predefine a shape as it's eaiser to use that shape instead of defining Hepha::Shape() multiple times. The next three parameters are (in this order), scale, position, and color. The last two parameters are image(texture), and interpolation mode, if you have a small image, say 16 x 16, I recomend you use Hepha::Texture::CLOSEST, else use Hepha::Texture::LINEAR.
